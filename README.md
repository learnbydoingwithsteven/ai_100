## 🚀 Quick Start

### New Full-Stack Structure

Each application now has a complete full-stack architecture:

```
app_XXX/
├── frontend/           # React + TypeScript
│   ├── src/
│   ├── public/
│   └── package.json
├── backend/            # FastAPI + Python
│   ├── app/
│   ├── tests/
│   └── requirements.txt
├── docker-compose.yml  # Multi-container orchestration
├── .github/workflows/  # CI/CD pipelines
└── docs/              # Documentation
```

### Running an Application (Full-Stack)

```bash
# Navigate to application
cd app_001

# Start all services (frontend, backend, database, cache)
docker-compose up

# Access the application
# Frontend: http://localhost:3000
# Backend API: http://localhost:8000
# API Docs: http://localhost:8000/docs
```

### Legacy Monolithic Apps

Original Python scripts are still available in each app directory:
```bash
cd app_001
python app.py  # Run original monolithic version
```

## 🔗 Getting Started

### 1. Migrate Applications to Full-Stack

```bash
# Run migration script for all 100 apps
python migrate_to_fullstack.py
```

### 2. Run Individual Application

```bash
# Navigate to app
cd app_001

# Start with Docker Compose
docker-compose up

# Or run legacy version
python app.py
```

### 3. Deploy to Production

```bash
# Build and deploy
docker-compose -f docker-compose.prod.yml up -d

# Or use Kubernetes
kubectl apply -f k8s/
```

### 4. Browse All Applications

```bash
# Use master launcher
python launcher.py
```
